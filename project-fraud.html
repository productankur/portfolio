Engineering Decisions
Why Each Choice Was Made

Every major technical decision was data-driven — not assumption-based.

⚡
XGBoost over Random Forest
XGBoost achieved AUC 0.9923 vs RF's 0.9892. More importantly, it scored higher recall on fraud cases — the metric that matters most in financial fraud detection.
✂️
Model Segmentation
Fraud in TRANSFER (0.77% rate) behaves differently from CASH_OUT (0.18% rate). Specialist models improved TRANSFER recall to 100% and reduced false alarms by 99%.
⚖️
SMOTE for Imbalance
Only 0.13% of transactions are fraud. Without SMOTE the model would predict "legit" for everything and hit 99.87% accuracy while catching zero fraud.
🎯
Threshold Tuning
Default 0.5 threshold caused 29,634 false alarms. Tuning to 0.9829 using F1 maximisation reduced false alarms to 291 — a 99% reduction.
🔍
SHAP over Feature Importance
Feature importance shows WHAT matters globally. SHAP shows WHY a specific transaction was flagged — essential for compliance, auditability, and analyst trust.
🚫
Dropped isFlaggedFraud
The existing rule-based flag would have leaked the answer to the model — like an open-book exam. Removing it forces the model to learn real patterns independently.
📐
Feature: orig_balance_diff
Raw balance columns were 78% correlated. Combining them into a single "amount drained" feature became the strongest predictor at 67.7% SHAP importance.
🔬
Kept Outliers
In fraud detection, extreme transactions ARE the signal. Removing outliers would delete the most suspicious cases. IQR flagging was used instead of removal.
